What is MIS?

MIS = organizational systems that leverage technology, human resources, and processes to systematically collect, process, store, and distribute information to support decision-making and control in an organization.

3 pillars of MIS: Technology + People + Processes → Information → Decisions

Historical Evolution

EraDevelopment
1940s–50sPunch card data processing; UNIVAC I (first commercial computer)
1960sMainframes; Decision Support Systems (DSS) emerge
1970sDBMS introduced; more organized data management
1980sPCs widespread; Microsoft Windows (1983); user-friendly interfaces
1990sERP systems (SAP); Internet changes architecture; relational databases
2000sCloud computing; mobile technologies; e-commerce business models
2010s+Big Data; AI/ML integration; IoT; real-time analytics

Role of MIS in Organizations

  • Streamline operations — automate routine tasks
  • Decision support — provide relevant, timely information
  • Strategic planning — align technology to organizational goals

Week 2 — Systems Approach to Organizations

What is a System?

A collection of interrelated components working together toward a common goal. Systems thinking views problems through multiple perspectives by examining relationships between elements.

System = INPUTS → TRANSFORMATION PROCESS → OUTPUTS
Open Systems
Exchange information, energy, or material with the environment. Have permeable boundaries. Example: Organizations, biological systems.
Closed Systems
Do not interact with environment. Rigid, impenetrable boundaries. Rare in practice. Example: Some mechanical systems.
Open Loop Systems
No feedback mechanism. Output does not influence input. Example: A toaster (no temperature sensor).
Closed Loop Systems
Have feedback. Output is fed back as input to self-correct. Example: Thermostat, air conditioning system.

Organization as an Open System

Inputs: People, money, materials, information from environment
Processes: Management functions (Planning, Organizing, Leading, Controlling)
Outputs: Products, services, information back to environment
Feedback: Customer satisfaction, market data → adjusts processes

Org Sub-systems: Accounting · Marketing · HR · Operations · IT · Finance

Role of Management (Control Mechanism)

Management acts as the CONTROL MECHANISM of the organizational system.

  • Planning — Set objectives and strategies
  • Organizing — Arrange resources and tasks
  • Leading — Direct and motivate people
  • Controlling — Monitor performance, compare to plans, correct deviations
MIS supports all 4 management functions by providing information at each stage.

What is a KMS?

An integrated set of processes and technologies used to capture, store, organize, retrieve, reuse, and create new knowledge to improve decision-making and organizational performance.

Knowledge = economic asset. Without a KMS, critical knowledge walks out the door when employees leave.

Classic example from lecture: "John knows everything about the 40-year-old system. He's about to retire." → This knowledge loss is exactly what KMS prevents.

Types of Knowledge

Tacit Knowledge
Experience-based, hard to write down. Leadership skills, intuition, expertise. Most valuable, hardest to capture.
Implicit Knowledge
Not documented yet but transferable if asked. Exists in people's heads but can be explained.
Explicit Knowledge
Documented and easy to share. Manuals, reports, whitepapers, SOPs. Easiest to manage.

The 5 Components of a KMS

Creates, manages, and publishes content — web pages, articles, manuals, policies. Example: WordPress, SharePoint.
D
Document Management System (DMS)
Stores and tracks documents with version control, access permissions, and archiving. Example: Google Drive, SharePoint document library.
D
Data Warehouse
Central repository for structured historical data from multiple sources. Uses ETL (Extract, Transform, Load). Supports analytics and reporting.
A
AI / ML / LLM Tools
Intelligent analysis, pattern recognition, natural language processing, automation. Turns KMS from a storage system into an intelligent assistant.
S
Security / Governance / Visualization Tools
Access controls, compliance enforcement, data privacy. Dashboards and visual reporting to present insights.

Week 7 — Enhancing Decision Making Through AI-powered MIS

⭐ Dr. Ruwan's second exam question. The confusion matrix calculation is almost certain to appear.

What is a Machine Learning Model?

A computer program that learns patterns from data to make predictions or decisions without being explicitly programmed. It improves with more data and experience.

Types of ML — Supervised vs Unsupervised

Supervised Learning
Trained on labeled data (each input has a known correct output). Model learns to predict output for new inputs.

Use cases: Spam detection, credit scoring, disease prediction, customer churn prediction
Unsupervised Learning
Trained on unlabeled data. Model discovers hidden patterns/groups on its own.

Use cases: Customer segmentation, anomaly detection, topic modeling, market basket analysis
Quick test: Does your training data have correct answers labeled? YES → Supervised. NO → Unsupervised.

Classification vs Regression (both are Supervised)

Classification
Predicts a category / class (discrete output)

Examples: Spam/Not spam · Disease Yes/No · Loan Approved/Rejected · Customer will churn/not churn
Regression
Predicts a continuous number (numeric output)

Examples: House price · Sales forecast · Temperature · Age
Quick rule: Answer is a category/label → Classification. Answer is a number → Regression.

The ML Workflow — 7 Steps

"Do People Create Features, Train Every Model?"
Define → Prepare → Clean → Features → Train → Evaluate → Model/Deploy
1
Problem Definition
What to predict? Classification or regression? Define success metrics. Understand the business objective.
2
Data Collection
Gather from databases (SQL/NoSQL), APIs, logs, documents, images, public datasets (e.g., UCI Repository). More quality data = better model.
3
Data Preprocessing / Cleaning
Handle missing values (remove or impute), remove duplicates, fix errors, normalize/standardize numeric data, encode categorical variables (one-hot encoding).
4
Feature Selection / Engineering
Choose the most relevant input variables. Remove irrelevant or redundant features. Create new features if needed. Use EDA (Exploratory Data Analysis) to understand distributions.
5
Model Selection & Training
Choose algorithm (Decision Tree, Naive Bayes, etc. in Weka). Split data: Training set (fit model) + Test set (evaluate). Use cross-validation (k-fold) for better accuracy estimates. Hyperparameter tuning: Grid Search, Random Search.
6
Model Evaluation
Test on unseen data. Use confusion matrix → Accuracy, Precision, Recall, F1 Score. For regression: MSE, RMSE, MAE.
7
Deployment & Monitoring
Deploy model to production. Monitor performance over time. Retrain as new data arrives. Explainability: SHAP, LIME, Feature importance.

⭐ Confusion Matrix & Metrics — HIGH EXAM PRIORITY

Predicted: YES (Positive)Predicted: NO (Negative)
Actual: YESTP — True PositiveFN — False Negative
Actual: NOFP — False PositiveTN — True Negative
Accuracy = (TP + TN) / (TP + TN + FP + FN) ← "Overall correctness"
Precision = TP / (TP + FP) ← "Of predicted YES, how many were correct?"
Recall = TP / (TP + FN) ← "Of actual YES, how many did we find?"
F1 Score = 2 × (Precision × Recall) / (Precision + Recall) ← "Balance of P and R"
Sensitivity = Recall = TP / (TP + FN)
Specificity = TN / (TN + FP) ← "Of actual NO, how many correctly identified?"
G-Mean = √(Sensitivity × Specificity) ← From past paper 2023/24 Q4b!
Memory trick: Precision = "predicted positive" denominator (TP+FP). Recall = "real positive" denominator (TP+FN). TP is ALWAYS the numerator for both.

AspectDetails

In Weka's confusion matrix: rows = actual class, columns = predicted class. Diagonal values = correct predictions.

Domain Knowledge in ML (from past paper Q1e)

Why domain knowledge matters in building ML models:

  • Feature selection — A doctor knows which symptoms are diagnostically relevant; a data scientist alone may miss this
  • Data interpretation — Understanding what "normal" looks like (e.g., blood pressure ranges)
  • Avoiding spurious correlations — Domain experts catch statistically significant but meaningless correlations
  • Model validation — Experts can judge if predictions make clinical/business sense
  • Feature engineering — Creating meaningful derived features that a domain expert knows matter
Example: To predict customer churn in telecom, a domain expert knows that "contract type" and "tenure" are more important than "zip code" — even if the data shows a correlation for zip code.

Week 12 — Introduction to DBMS

What is a DBMS?

Sophisticated software that controls and manages a collection of related data, ensuring it can be shared, updated, and retrieved efficiently and securely.

Database = the data. DBMS = the software that manages it (e.g., MySQL, Oracle, PostgreSQL, Microsoft SQL Server).

Advantages of the Database Approach

  • Encourages data sharing across departments
  • Controls data redundancy (no duplicate data in multiple files)
  • Improves data accuracy and integrity
  • Permits storage of vast data volumes with efficient access
  • Allows database queries (SQL)
  • Controls data security, privacy, backup and recovery
  • Enforces standards (naming, formats)

ACID Properties — Database Integrity

ACID = Atomicity · Consistency · Isolation · Durability
PropertyMeaningExample
AtomicityTransaction is all-or-nothing. Either fully completes or fully rolls back.Bank transfer: debit AND credit must both succeed, or neither happens.
ConsistencyTransaction brings database from one valid state to another. Rules/constraints always satisfied.Account balance can't go below 0 after transaction.
IsolationConcurrent transactions don't interfere with each other.Two users booking the last seat simultaneously — one gets it, one gets an error.
DurabilityOnce committed, transaction persists even if system crashes.After "Payment confirmed," the record stays even if server crashes.

Three-Schema Architecture

External Schema (View Level)
What each individual user/application sees. Customized views hiding complexity.
Conceptual Schema (Logical Level)
Overall structure of the whole database — entities, relationships, constraints. Independent of storage.
Internal Schema (Physical Level)
How data is physically stored on disk — indexes, storage structures, file organization.

DBMS Languages

DDL — Data Definition Language
Define database structure. CREATE TABLE, ALTER TABLE, DROP TABLE.
DML — Data Manipulation Language
Manipulate data. SELECT, INSERT, UPDATE, DELETE.

Database Users

  • Database Administrator (DBA) — manages the DBMS, security, backups, user accounts
  • Database Designers — design the schema
  • End Users — Casual (occasional queries), Naïve/Parametric (regular transactions), Power users (complex queries)
  • System Analysts & Programmers — build applications using the database

Cloud vs In-house Database (from past paper Q5a)

Cloud-hosted Database
Advantages:
  • Scalable (grow as needed)
  • Lower upfront cost (pay-as-you-go)
  • Provider handles maintenance/updates
  • Built-in disaster recovery
  • Access from anywhere
Disadvantages:
  • Ongoing subscription cost
  • Data sovereignty/privacy concerns
  • Dependent on internet connectivity
  • Less control over security policies
In-house Database
Advantages:
  • Full control over data and security
  • Can customize to specific needs
  • No recurring cloud fees
  • Data stays on premises
Disadvantages:
  • High upfront hardware cost
  • Need internal IT expertise
  • Harder to scale quickly
  • Responsible for own backup/recovery

Week 14 — Cybersecurity & Social Engineering

What is Cybersecurity?

Protection of information systems (hardware, software, infrastructure), the data on them, and services they provide from unauthorized access, harm or misuse.

Social Engineering — EXAM SCENARIO (Q6 past paper)

Social Engineer = attacker who uses psychological manipulation to trick people into divulging confidential information or taking harmful actions. Exploits human psychology, not technical vulnerabilities.

6 Psychological Triggers Exploited:

Authority
Pretending to be an executive or IT manager to get compliance.
Liking
Building rapport and friendship before making a request.
Reciprocation
Doing a "favour" first, then asking for something in return.
Consistency
Getting verbal commitment to a fake policy, then leveraging it.
Social Validation
Claiming "your colleagues already did this" to build trust.
Scarcity
Creating urgency ("first 10 people win a prize!").

Phishing — Types

  • Email Phishing — Fake email with malicious link or attachment
  • Vishing — Voice phishing via phone call
  • Smishing — SMS phishing with malicious link
  • Spear Phishing — Targeted at a specific individual with personalized details
  • Whaling — Targeting senior executives (C-suite)

Red Flags in Phishing Emails (from past paper Q6c):

  • Unsolicited "you've won" messages
  • Urgency and pressure to act immediately
  • Request for sensitive personal/financial information
  • Suspicious links (hover to check real URL)
  • Too-good-to-be-true offers
  • Generic greetings ("Dear Customer") or mismatched sender addresses

Types of Malware

"Very Sick Trojans Run Wild Rampaging" = Virus · Spyware · Trojan · Ransomware · Worm · Rootkit
TypeHow it works
VirusInserts itself into programs; executes when app runs; spreads through files
WormSelf-replicates and spreads across networks without needing a host program
TrojanAppears legitimate but contains malicious code; doesn't self-replicate
SpywareHides on device, monitors activity, steals passwords/financial data
RansomwareEncrypts victim's files, demands payment for decryption key
RootkitHides at system level, provides persistent unauthorized access

Identity Theft

  • Financial — Using stolen identity to take loans, credit cards
  • Medical — Posing as another to obtain medical care
  • Criminal — Using another's identity during arrest to avoid record

Ways information is collected for identity theft:

  • Hacking into computers/networks
  • Information-gathering malware
  • Searching hard drives of discarded computers
  • Phishing and social engineering
  • Physical theft of documents/mail

Cybersecurity Measures (for organizations)

Technical Measures
  • Multi-factor authentication (MFA)
  • Encryption (data at rest and in transit)
  • Firewalls and intrusion detection systems
  • Regular security patches and updates
  • Fraud detection using ML algorithms
  • Backups and disaster recovery plans
Organizational Measures
  • Security awareness training for all staff
  • Clear policies for handling sensitive data
  • Strict access controls (principle of least privilege)
  • Regular security audits
  • Incident response plan
  • Culture of trust and open reporting

Q1e (2 marks) — Domain knowledge importance in ML models
Domain knowledge is critical because: A data scientist alone may not know which features are diagnostically significant. Example: In a medical ML model predicting heart disease, a cardiologist knows that "chest pain type" and "cholesterol level" are far more important than "patient ID" or "zip code." Q4b (3 marks) — G-Mean calculation from classification result table (15 sentences)
From the table: Actual: P P N P P P N P N P N N P P P | Predicted: P P P P P P P P P P P P P P P

Count: TP=10 (actual P, predicted P), FN=0, FP=5 (actual N, predicted P), TN=0

Sensitivity (Recall) = TP/(TP+FN) = 10/10 = 1.00
Specificity = TN/(TN+FP) = 0/(0+5) = 0.00
G-Mean = √(1.00 × 0.00) = √0 = 0.00

Note: A G-Mean of 0 indicates the model never predicts negative class — it always predicts P. This reveals severe class imbalance handling failure.
Q6 (2 marks each) — ABC Corp Phishing Scenario
a. Social engineer & psychological manipulation: A social engineer is a cybercriminal who exploits human psychology rather than technical vulnerabilities. In the ABC case, the attacker used authority (corporate branding) and excitement/greed (prize of $1M) to bypass the employee's critical thinking, making them act without verifying.

b. Steps to verify email legitimacy: (1) Contact company directly using official numbers (not numbers in the email); (2) Check sender email address carefully for spoofing; (3) Hover over links to see real destination URL; (4) Consult IT security team before clicking.

c. Three red flags: (1) Unsolicited prize notification — company-sponsored contest employee never entered; (2) Request for sensitive personal data (NIC, bank account) through a form; (3) Link redirects to unfamiliar/unofficial website even if it appears legitimate.

d. Consequences: Employee: identity theft, financial fraud, personal distress, reputation damage. ABC Corp: corporate espionage (stolen internal documents), reputational damage, regulatory penalties, financial losses, loss of client trust.

e. Organizational policies: Regular security awareness training; mandatory email verification procedures; multi-factor authentication; strict access controls; incident reporting culture.

f. Malware types: Malware = malicious software designed to damage or gain unauthorized access. Types: Virus (inserts into programs), Worm (self-replicates across networks), Ransomware (encrypts files, demands payment).
